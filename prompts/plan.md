### Decisions taken (were open)

- **API shape: expression string** (`POST /api/v1/calculate` `{"expression": "..."}`), settled by the UI spec — one big free-text input. Shunting-yard tokenizer→RPN→eval in Go; all math semantics server-side.
- **√ is a unary prefix operator**, not a paren-requiring function. Backend accepts `√` and `sqrt`, parens optional. Frontend: keypad `√` inserts `√()` cursor-inside, typed `sqrt` collapses to `√`, unclosed parens auto-balance on submit, `)` skips over existing closers.
- ~~No MathJax/KaTeX~~ **OVERRIDDEN 2026-07-20 (Abror): MathLive `<math-field>` replaced the free-text input.** LaTeX is source of truth; MathLive owns cursor/formula handling, virtual keyboard off, keypad inserts into the field. New isolated engine module `src/engine/translate.ts`: LaTeX → MathJSON (`@cortex-js/compute-engine`, **canonical: false** — canonical computes results and folds 1/0 to ComplexInfinity) → whitelisted walk emitting exactly the expr.go grammar; anything else → "unsupported: X" client-side, no request. `src/lib/input.ts` (sanitize/cursor/paren helpers) retired — MathLive does it natively. Backend unchanged. Notes: `%` resolves at CE parse time (50\% → 0.5, but only attached to a literal); CE normalizes `2++3` to Add (e2e invalid case now `2+`); MathLive serializes brace-less single args (`\sqrt9`, `\frac10`); MathLive has an internal role=status announcer — e2e targets `getByRole('status', {name: 'result'})`; typed e2e flows need `click` + `toBeFocused()` gate before `keyboard.type`.
- Errors: 400 `invalid_request` (transport/shape), 422 `invalid_expression`/`division_by_zero`/`overflow`/`undefined_result` (math). JSON can't carry Inf/NaN → 422s.
- Embed via `-tags embed` build-tag pair (embed can't cross module dirs; Makefile copies dist into backend/). `make run` = rebuild + run binary.
- All hosts/ports from `.env` incl. BACKEND_HOST; configs fail fast when unset; Dockerfile carries its own ENV.

- **Keypad-close flicker (fixed 2026-07-20)** had three stacked causes, found via CDP screencast frame-diffing: (1) backdrop-filter on the keypad — snapshots can't capture it, stale box; (2) keypad in normal flow reflowed the stage → moving snapshot resampled the 1px underline gradient (dim→pop). Keypad+toggle are now fixed overlays, nothing reflows; (3) CSS transitions (underline focus gradient via toggle stealing focus, toggle color) freeze during a view transition and visibly replay after it ends — toggle now preventDefaults pointerdown and has no transition property. All VT durations unified at 220ms. Verified: post-VT frames differ only by caret blink (~50dB PSNR on teardown frame).
- Bundle: vendor chunks split (mathlive 801k, compute-engine 1.84M gz 497k) via rolldown `codeSplitting` (advancedChunks deprecated); warning limit 2000k.
- **Frontend architecture (Abror directive 2026-07-20): no state in components.** zustand store `src/store/calculator.ts` (state + actions incl. the field element ref), `src/hooks/useMathField.ts` (element wiring via React 19 ref-callback cleanup), presentational components in `src/components/`. Store actions are stable references → keypad no longer re-renders on `=`. zustand chosen over Context for later localStorage history via persist middleware.

- **History (done 2026-07-20):** recorded on successful submit only; `{hash, latex, result, at}`, dedupe by 8-hex FNV-1a of latex (`src/lib/hash.ts`) moving repeats to top, cap 50. Persisted via zustand persist (partialize → history only; in-memory storage fallback keeps node tests quiet). UI: second dock toggle (clock) + bottom sheet mirroring the keypad (panels exclusive via `panel` state); entries rendered with `convertLatexToMarkup` + `mathlive/static.css`; click → recall (re-input latex, close panel). Abror's rationale: bottom-dock pattern scales to mobile.

- **Security review round (2026-07-20, Abror's findings):** (1) Dockerfile → distroless `:nonroot` ✓; (2) securityHeaders middleware (nosniff, CSP, referrer) — CSP needs `style-src 'unsafe-inline'` AND `font-src data:` for MathLive (found via browser test against embedded binary; header-only tests would have missed it) ✓; (3) /assets/ listing suppressed in static_embed.go ✓; (4) convertLatexToMarkup/innerHTML self-XSS = accepted risk, comment in HistoryPanel (Abror added) ✓. Abror also added http.Server timeouts.
- **Examples feature:** named formula chips (pythagoras, compound growth, tip %, float trap, nested roots, power tower) float in free space around the field (fixed-position desktop, flow row <980px), hover `title` explains semantics, click → store.setFormula (recall now reuses it). Rendered via convertLatexToMarkup.

- **Examples v2:** hideable via third dock toggle (scatter icon); each reopen bumps `scatterSeed` → `src/lib/scatter.ts` (mulberry32 PRNG, deterministic, unit-tested) jitters slot positions/tilt/drift timing. Cards carry 2-sentence explanations (hidden <980px). Slow "living" drift via `translate` keyframes + per-card `--drift-dur/-delay` vars, `rotate` kept separate so they don't clash; disabled under prefers-reduced-motion. `examplesOpen` persisted.
- **VT overlay ate clicks (Abror report):** during any view transition the `::view-transition` pseudo covers the viewport and swallows pointer events (cursor flips to default, dock buttons dead ~220ms). Fix: `::view-transition { pointer-events: none }`. Regression e2e: double-click scatter toggle back-to-back.
- **Playwright gotcha:** `use.reducedMotion` silently ignored in 1.61 — `page.emulateMedia({ reducedMotion: 'reduce' })` in beforeEach instead; also required because infinitely-drifting cards never pass Playwright's stability check.

- **Guided mode (Abror's "hold formula in mind" idea):** `engine/guides.ts` — MathJSON templates with `$slot` numbers, matched non-canonically on every input event (`syncGuide`); structure fixed, digits free (slot unification: repeated slots must agree). 8 guided patterns with computed steps; reactive SVG diagrams (triangle/bars/growth curve) in GuidePanel. Guide panel grows the stage downward → field rises from center (Abror preferred up over down/keypad-collision). VT only on guide appear/disappear, not per keystroke.
- **Formula catalog (Abror pivot):** ~40 formulas, 7 categories, colorized via `--cat-*` vars; library bottom-sheet (4th dock button) + floating cards became a seeded random sample of 8. `scatterPositions` v2 samples from free bands around minimal exclusion boxes (stage core + dock) — full-viewport spread. Every entry unit-tested through translateLatex; guides' latexes asserted present in catalog.
- **Gotchas this round:** `.stage{z-index:1}` (card layering) put the transparent stage box over the fixed keypad and silently swallowed key clicks — panels/dock now z-index 2 (found via Playwright "visible, enabled and stable" then hang = covered element). Playwright default role-name matching is substring — keypad '7' matched a card named "…0.7…"; use exact.

- **Guide expansion via 4 parallel Opus subagents (2026-07-20):** each probed real MathJSON before writing templates; deliverables to scratchpad, integrated + SVG-reviewed by hand per Abror's mandate. Now 36 guides, 17 diagrams (4 data-spec + 13 bespoke components under `DiagramProps` contract in components/diagrams.tsx registry). Agent quality was high on templates/pedagogy (pinned literals for 1/3 & 355/113 to avoid hijacking typed fractions; convexity accept for weighted average; exact backend codes read from expr.go for edge-case guides). Review caught: label clipping (e-convergence at small n, big-tiny axis endpoints), silent fractional-n rounding, off-contract fill, guide-text column not shrinking (real clip), golden-ratio strip label overflow, `1e-15` rendering as "1e−15" in latex (show() now emits ×10^n). Decisions: markup+sales tax structurally identical → merged as 'scale up by a rate'; rectangle area & average speed left unguided (would mislabel any int·int / a/b). Ghost "bug" in review screenshots was actually mid-flight view transitions — added prefers-reduced-motion kill for VT animations (a11y win).
- **2^k collision:** chessboard (Power(2,$k)) sits before cube volume (Power($a,3)) → 2^3 shows the chessboard guide. Documented, acceptable.

- **Final decomposition (Abror directive):** `engine/guides.ts` (~950 lines) → `engine/guides/` (types, helpers/show, basics/curiosities/geometry/algebra/applied guide data, matcher+ordered assembly in index — order is load-bearing, first match wins). `components/diagrams.tsx` → `components/diagrams/` (one file per SVG component, SpecDiagram renderer, shared plane.tsx grid, registry in index). Guide panel widened to 780px, diagram column 260px (step latex was overlapping the SVG). History cap already existed (HISTORY_LIMIT=50).
